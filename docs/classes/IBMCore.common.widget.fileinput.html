IBMCore.common.widget.fileinput Class

Defined in: src/js/sources/ibm/common/widgets/fileinput.js:1
Module: IBMCore

File input widget.

Automatically creates the file input widget onload based on the presence and value of the magic @data-widget="fileinput" attribute on the tab container. And optional "data-multiple" setting is available but your server-side processing must be able to support it (array parameters).

Usage:

<input id="myInputField1" type="file" data-widget="fileinput" />

Event usage: Subscribe to the IBM widget init event that fires on your DOM element to know when the widget has been initialized and you can access its APIs.

Special case: In scenarios where you dynamically inject HTML post page load, you can initialize the widget on your injected container like this:

jQuery("<your_input_selector>").fileinput();

To destroy the widget and return back the original HTML that the widget initialized on:

jQuery("<your_input_selector>").data("widget").destroy();

Allowed options and values. All values are strings (html standards):
data-multiple: true | false (default)   // Allows multiple attachments
data-translation-browse: "string_value"   // Provide a localized translation for "Browse"
data-translation-remove: "string_value"   // Provide a localized translation for "Remove". Only used be screen readers, not visible.
