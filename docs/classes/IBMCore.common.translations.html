IBMCore.common.translations Class

Defined in: src/js/sources/ibm/common/util/translations.js:1
Module: IBMCore

Automatically gets the v18 common translations for all the UI text and links.

Automatically runs onload. Waits until meta fires dataReady event, then gets the page locale's translation files, then publishes event for subscribers.
Any time that the meta init is run and publishes "dataReady", this translation init will run then publish "dataReady".

This uses the "data" utility. The data file calls the "provide" and automatically puts the translation object as a named child in translation.data. This uses 'promises' to work.

init

()

Defined in src/js/sources/ibm/common/util/translations.js:49

Callback from subscription to meta "dataReady" event publisher.
Validates cc and lc, etc. If translations exist in storage, uses cached copy and passes them to callback, else fetches translation data file for the locale.
Note: When this runs and publishes ready event, it calls all active subscribers' callback functions.

storeTranslations

() private

Defined in src/js/sources/ibm/common/util/translations.js:154

Adds the current locale translations (me.data) into currStoredTranslations{} then saves into localStorage.
Only stores the latest one, otherwise it builds up with locale change.
Current TTL for stored translations is 4 hours.
