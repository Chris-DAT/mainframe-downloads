IBMCore.common.widget.syntaxhighlighter Class

Defined in: src/js/sources/ibm/common/widgets/syntaxhighlighter.js:1
Module: IBMCore

Syntaxhighlighter plug-in pre-processor.

Sets defaults for the Syntaxhighlighter plug-in, cleans it up and other random processing to make the widget work.
Then it simply inits the jQuery Syntaxhighlighter plug-in.

Usage:

<pre data-widget="syntaxhighlighter">...</pre>

Special case: In scenarios where you dynamically inject the HTML after page load, you can initialize the widget on your injected HTML like this:

jQuery("<your_pre_container_selector>").syntaxhighlighter();

To destroy the widget and return back the original HTML that the widget initialized on:

jQuery("<your_pre_container_selector>").data("widget").destroy();

Styling options are set via the class attribute. All values are strings (html standards):
data-maxheight : {Any valid CSS max-height value} The max-height setting to use to enable vertical scrolling of the container.
class="brush: js; html-script: true"   // (default if no class is set) Format for HTML and JS inside <script> tags.
class="brush: js"   // Format for inline/pure JS code only.
class="brush: js; html-script: true; gutter: true"   // Show the gutter and line #s.
class="brush: js; html-script: true; gutter: true; highlight: [4]"   // Highlight the 4th line in the code.

The highlighter style accepts an array of 1 or more #s, or a range. All of the following are valid:
highlight: [4]
highlight: [2,4,7,13]
highlight: [3-7]

This uses the syntaxhighlighter plug-in. View the documentation for full documentation and APIs available.
