Element classes

  • ibm-data-table
    Styles the data table properly.
  • ibm-grid
    Adds cell borders to the table, giving it a grid/spreadsheet type style.
  • ibm-no-highlight
    Removes the default blue highlight on row hover.
  • ibm-padding-small
    Reduces the default padding in each cell. Best used on large data set tables that you want to compress.
  • ibm-altrows
    Colors every other row a light gray.
  • ibm-altcols
    Colors every other columns a light gray.
  • ibm-bgcolor-blue-10
    Add to a <td> to color the background light blue, with a darker shade on hover.
  • ibm-bgcolor-green-10
    Add to a <td> to color the background light green, with a darker shade on hover.
  • ibm-bgcolor-magenta-10
    Add to a <td> to color the background light magenta, with a darker shade on hover.
  • ibm-bgcolor-orange-10
    Add to a <td> to color the background light orange, with a darker shade on hover.
  • ibm-bgcolor-purple-10
    Add to a <td> to color the background light purple, with a darker shade on hover.
  • ibm-bgcolor-red-10
    Add to a <td> to color the background light red, with a darker shade on hover.
  • ibm-bgcolor-teal-10
    Add to a <td> to color the background light teal, with a darker shade on hover.

Official standards & guidelines

ibm.com/standards/web/design/data-tables/


What

A data table is a visual instrument comprised of labeled columns and rows, used to arrange information. It may serve to organize disparate data, as well as to permit data to be easily manipulated and analyzed.


Why

When you have large sets of data to show users or a want to show data comparison between sets of data, a data table allows you to present numerical data inside of a grid format.


What's new

Data tables style options have been redesigned for v18. New classes are available for alternating columns, rows, colors and a new "grid" style options.


Rules & guidelines

  • All data tables require the class "ibm-data-table".
  • Ensure tables have proper caption, head and body elements, else it will not look right.
  • Do not use tables for layout. They should only be used to display content or data in a grid format.
  • For advanced features like pagination, filtering, sorting, scrolling and more, use the datatable widget.
  • You cannot use both alternating rows and alternating columns.

How

Default IBM styled tables

  • Code your table using all normal, proper, standard HTML, nothing special.
  • Add the required class "ibm-data-table" to the <table> for proper default styling.

Alternating row background colors

  • Code your table according to the default IBM styled table rules. In case you forgot, it's the first bulleted list in this "how to" section.
  • Add the class "ibm-altrows" to the <table>.

Alternating column background colors

  • Code your table according to the default IBM styled table rules. In case you forgot, it's the first bulleted list in this "how to" section.
  • Add the class "ibm-altcols" to the <table>.

Full borders/"grid" styled table

  • Code your table according to the default IBM styled table rules. In case you forgot, it's the first bulleted list in this "how to" section.
  • If you want to make your table look like a spreadsheet, add the class "ibm-grid" to the <table>.

Colored/highlighted cells

If you want to highlight or call attention to particular cells by giving them a background color, you can do so but you must use these standard classes. Add one of classes to the <td> that you want to highlight:

  • ibm-bgcolor-blue-10
  • ibm-bgcolor-green-10
  • ibm-bgcolor-magenta-10
  • ibm-bgcolor-orange-10
  • ibm-bgcolor-purple-10
  • ibm-bgcolor-red-10
  • ibm-bgcolor-teal-10
  • ibm-bgcolor-yellow-10
