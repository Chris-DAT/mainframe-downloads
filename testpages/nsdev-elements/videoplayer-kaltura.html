Official standards & guidelines

ibm.com/standards/web/design/video/


What

The video player is a JS widget that will dynamically and automatically embed a video into your web page so that a use can watch it without going offsite or off page.


Why

Video is an integral part of our lives, and it's use online is pervasive. Video can provide a passive engagement medium where visitors can experience your message with very little effort. Also, people are lazy and prefer to watch vs. read.


What's new

The video player has been completely revamped for v18 and now also supports videos hosted and managed in Kaltura. The settings, rules, configurations, etc are all the exact same as the Youtube video player except for the "videotype" param. In other words, you only need to learn one set of options and they can be applied to the video player no matter what the video "source" is.


Rules & guidelines

  • Do not autoplay. Usability guidelines advise against autoplay as it is an interruption technique that annoys people and makes them click the back button.
  • Use video for a reason. Don't just add it to your pages because you can. Treat video like imagery and ask yourself "is this adding value to my message?"
  • Remember that not everyone can hear, nor see. If your video includes sound you must include a transcript and closed-captions with your video to be accessibility compliant.
  • There are lot of other guidelines about using video. Like everything else... just be smart about it.
  • Ideally use progressive enhancement and make a link that links to Kaltura in the case that JS is off or broken. Then if the user has JS turned on, the widget will convert the link to the video player widget.

How

NOTE: The options for displaying the video widget are the same as Youtube (inline, overlay, title placement, etc) except for custom YT stuff which isn't available on Kaltura players.

Embedding a video is very easy. You can play a video "inline" in your page, or have it play in an overlay.

General

  • A video player widget can be configured as a single video or a carousel of videos via a simple CSV list of video IDs.
  • Playlist widgets can be set to play the videos in an overlay or inline (in the carousel).
  • View the video player JS documentation for a full list of supported options and APIs available.

Overlay players

  • You can make a text link or an image link be a trigger for an overlay player.
  • You can optionally have the video duration automatically added to your link if you are using a text link as the widget/overlay trigger.
  • If using custom JS or need to trigger a video programmatically, you can show the video player overlay and start playing it using the available APIs
  • View the video player JS documentation for a full list of supported options and APIs available.

Inline players

  • An inline player can use and display the thumbnail associated with the video from Kaltura, or you can code your own inline image to use.
  • If using a custom image, you can optionally have the play button overlaid on top of your image.
  • You can optionally show the video title, description, duration, either below or overlaid on top of the placeholder image, in any combination.
  • Inline players will work in any size column and automatically resize with the grid and a user's screen size.
  • If using custom JS or need to trigger a video programmatically, you can play the video and control it using the available APIs
  • View the video player JS documentation for a full list of supported options and APIs available.

Sample popular data attribute settings

  • data-videodisplay: Sets how to play the video, in an overlay, inline or carousel with either.
  • data-showvideoduration: Allows you to hide the video duration (shown automatically).
  • data-showvideotitle: Allows you to hide the video title information.
  • data-videotitleoverride: Allows you to specify an alternate video title to display.
  • data-customplaceholder: Allows you to use a custom image instead of the default video thumbnail.
  • data-showimageplaybutton: Allows you to show the video play button over your custom image placeholder.
  • data-carouselcolumns: Set the number of columns to show per carousel slide. AKA: ibm-col-X-1s
  • View the video player JS documentation for a full list of supported options and APIs available.

Metrics events

Event that fires when a video's state changes (played, paused, etc):

  • ibmEV:video player - kaltura
  • ibmEvAction:$player_state  (what the player did with the video; played | paused | ended | error)
  • ibmEvGroup:$video_duration  (the length of the video, in seconds, also used in 'ibmEvVidLength')
  • ibmEvName:$video_title
  • ibmEvModule:$current_time  (the current timestamp of the video when the event happened, in seconds, also used in 'ibmEvVidTimeStamp')
  • ibmEvSection:##%  (the percentage of video time elapsed, simply; current time / video duration)
  • ibmEvVidStatus:$cm_state  (special coremetrics video state, associated to player state; 0 | 1 | 2 | 3)
