All modules IBMCore.common.module.xxxxx have a common convention of dis/enablement. All standard shell page modules can be enabled and disabled the same way.


Using the standard config utility

			<script>
				IBMCore.common.util.config.set({
					__MODULE_NAME__: {
						enabled: true|false
					}
				});
			</script>

Using that as a template, you replace the module name with the one you want to enable or disable, and then you set either true or false accordinly.


Using the DDO (digitalData object

The DDO contains the page's meta data. For web apps that do async loading of JS, you can set the config settings in the DDO so that whenever you load in the v18 JS, the settings are there an ready to be used by the JS.

Note that the syntax is the same, and the object that you create is the exact same. It's just that instead of passing it to a function, you are putting it in the DDO in the ibm property/namespace section:

			<script>
				digitalData = {
			        page: {
			            category: {
			                primaryCategory: "____"
			            },
			            pageInfo: {
			                effectiveDate: "____",
			                expiryDate: "____",
			                language: "____",
			                publishDate: "____",
			                publisher: "IBM Corporation",
			                version: "v18",
			                ibm: {
			                    contentDelivery: "____",
			                    contentProducer: "____",
			                    country: "US",
			                    .......
			                    config: {
				                    __MODULE_NAME__: {
										enabled: true|false
									}
								}
			                }
			            }
			        }
			    };
			</script>
