Greeensock animation library

Official standards & guidelines

ibm.com/standards/web/design/animation/


What

The creation of artificial moving images.


Why

They can be effective when used correctly; from leading the user, to a visual queue that something happened, to an entire immersive experience.


Rules & guidelines

Remember when Flash came out and everyone wanted to use it "just because". Don't make animation the new Flash. Use animation when appropriate. Does it solve a problem? Does it improve the experience? If there's no reason to use it... don't.

Animations do not stop when out of view/off screen. Be cautious of using multiple animations that continuously run. Consider disabling animations on small screens or when out of view to prevent overloading the user's computer.


How

Animation can be accomplished with CSS3, JS or a combination of both. CSS3 animation is the best solution if you want to animate few elements for simpler 'one-shot' transitions (such as scroll icon or fade-in effect on an image). But if you want more complex or advanced animations with more control of elements, we have the Greeensock animation library (GSAP) hosted and available for use on all IBM pages. If you use GSAP, read the documentation and view examples and tutorials on the Greensock web site. You are responsible for your own use of the library, support, troubleshooting, etc.

Greensock is more than just "static" animation. It has features/libraries for drag-and-drop also.
