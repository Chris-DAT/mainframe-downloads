Element classes

  • ibm-simple-show-hide
    Base class to use on the outer widget <div> container to setup a basic show/hide list of items with descriptions that are hidden.
  • ibm-show-hide-controls
    Properly formats the show/hide control container for the ibm-simple-show-hide.
  • ibm-show-hide
    Base class to use on the outer widget <div> container to setup a standard show/hide that uses the +/- heading as the trigger
  • ibm-container-body
    Use on a content <div> container immediately following a show/hide trigger element to denote it is the element that should be shown/hidden when the preceding trigger is clicked.
  • ibm-alternate
    Add to the outer container to make a bold version show/hide; bold text and thicker top border.
  • ibm-hideable
    Add to an item description container that you want hidden, for a simple show/hide.
  • ibm-twisty
    Add to the show/hide <ul> to make it a basic twisty show/hide list.
  • ibm-twisty-head
    Add to the <span> to set it as the twisty 'trigger'.
  • ibm-twisty-body
    Add to the <div> to set it as the element that the trigger should show/hide.

Official standards & guidelines

ibm.com/standards/web/design/show-hide/


What

Show/hide widgets allow you to hide non-essential content by default and let the user choose to discover it and show it's contents if they want to. It is a form of progressive disclosure.


Why

Show/hides and progressive disclosure help maintain the focus of a user's attention by reducing clutter, confusion, and cognitive workload. This improves usability by presenting only the minimum data required for the task at hand. However it comes at a price as the use may never notice or discover the content that is hidden.


What's new

Nothing except new style widget attributes.


Rules & guidelines

  • Never hide essential content. Only hide content you are OK with if the user never sees it.
  • Show/hides should be used sparingly. Making a user constantly click and interact with the page to simply read content is not the best UX. It's much easier to scroll than play mouse ninja and open and close a bunch of sections.
  • Show/hides are hidden by default.
  • Standard show/hide toggles include a top border.

How

There are several different types of standard show/hide widgets. Choose the one that best suits your needs:

  • Hide descriptions of items in a list.
  • A single basic show/hide toggle.
  • An accordion: A group of connected basic show/hides where only one is opened at a time and all others close.
  • List/Twisty/nested show/hides: Similar to a tree widget and windows file explorer style. Intended for basic content and supports limited styling options.

Metrics events

Event that fires when a standard show/hide section was shown or hidden, appropriate for the type of show/hide:

  • ibmEV:widget
  • ibmEvAction:$text  (the text of the show/hide trigger that was clicked)
  • ibmEvGroup:Showhide
  • ibmEvName:$event_type  (the event that happened; show | hide | show descriptions | hide descriptions)

Event that fires when a twisty (show/hide using nested ULs) is collapsed or expanded:

  • ibmEV:widget
  • ibmEvAction:$text  (the text of the show/hide trigger that was clicked)
  • ibmEvGroup:Twisty
  • ibmEvName:$event_type  (the event that happened; expand | collapse)
