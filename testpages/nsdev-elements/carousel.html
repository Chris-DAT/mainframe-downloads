Element classes

  • ibm-carousel__arrows--large
    Gives the carousel large arrows that bookend the carousel instead of under it. Can only be used on centered, non-full-width carousels, otherwise there is no room for the arrows next to the carousel.
  • ibm-carousel__controls--light
    Forces the carousel to have light colored controls. Use when your carousel is on a dark background.
  • ibm-carousel__controls--dark
    Forces the carousel to have dark colored controls. Use when your carousel is on a light background or your background conflicts with the default blue controls and do not meet required accessible color contrast ratios.

Carousel - With column splits

If you want to have multiple columns in each carousel panel, you can easily do so using the standard ibm-columns and ibm-col-x-x grid structure and coding standards. As well, carousel can be put inside any ibm-col-x-x and do no have to be full grid width. Carousels can be split into any # of columns using the standard grid structure and classes.

Carousel - With large arrows

A carousel doesn't have to be the full width of the grid. You can place it inside a ibm-col-x-x. When you do this with nested columns if you center the outer col using the ibm-center-block class you can use the "large arrows" option since you have available white space on the sides of the carousel.

The below carousel has each panel's content inside a ibm-col-12-9 ibm-center-block which gives the carousel extra room on each side to allow the use of the large arrows.

Official standards & guidelines

ibm.com/standards/web/design/carousels/


What

A carousel (sometimes called a "slider", and in v17 was known as a "ribbon") is a pattern for displaying content horizontally on a page, with a mechanism to allow the user to scroll left and right through the items in the carousel.


Why

A carousel optimizes screen space by displaying only a subset of images or items from a collection of items in a cyclic view and can be used when you do not have enough space to show all items at once. The navigational controls on a carousel suggests additional content that is not currently visible, this encourages the user to continue exploring. The carousel pattern can in this way be used as an extra incentive for the user to browse through all items of the list.


What's new

The HTML is very similar to v17, but can be simplified based on your usage. Carousels are truly responsive in v18 (not just set on page load) and will dynamically resize. A new set of APIs are available that make adding, removing, and controlling the carousel with JS easier than before.


Rules & guidelines

  • The carousel dots controls are enable by default.
  • The left and right arrow controls are optional but can only be used in a full grid width carousel (inside a 12-12 column). Otherwise the arrows bleed into the adjacent columns.
  • Dots or arrows, or both sets of controls can be enabled on a carousel.
  • Generally speaking they are not the best to use, and it's known from historical data that click-thru rates drop off exponentially with each item.
  • Use when you have a large set of items to show, but want to let the user concentrate their attention only on a select few items at a time.
  • Use when you want to tease the user by letting them know that there are more items available than what is currently shown.
  • Use when you do not have enough space to show all items at once.

How

The HTML is very similar to v17 and uses the standard v18 widget convention and requires the use of the data-widget="carousel" attribute to turn the <div> into the widget. A whole new set of APIs are also available that allow you to programmatically control the carousel including autoplaying, pausing, adding an item, removing an item, and more.

A very basic, bare minimum carousel looks like this:

		<div data-widget="carousel">
			
This is panel #1. This is panel #1. This is panel #1. This is panel #1. This is panel #1. 

			
This is panel #2. This is panel #2. This is panel #2. This is panel #2. This is panel #2. 

			
This is panel #3. This is panel #3. This is panel #3. This is panel #3. This is panel #3. 

		</div>

Hints and tips

  • Each immediate child element inside the widget <div> is a carousel "panel".
  • If you are not splitting each carousel panel into multiple columns, do you not need to use the grid in the carousel panels.
  • Dot controls are enabled by default. To disable them add the data-dots="false" attribute.
  • Arrow control are disabled by default. To enable them add the data-arrows="true" attribute.
  • Autoplay is disabled by default. To enable it add the data-autoplay="true" attribute.
  • To enable a random slide to be first add the data-randomfirst="true" attribute.
  • For an up-to-date listing of parameters, settings and APIs, view the carousel JS documentation.
