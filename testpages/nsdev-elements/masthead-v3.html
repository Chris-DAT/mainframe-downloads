Default

This is what every page gets by default. View this page with the default masthead.

Alternate

This is the default masthead but without the inline IBM categories (CAMS) links and megamenu. View this page with the alternate masthead.

Popup

This hides everything except the logo. Only use this on popup window pages. When the popup masthead is set, the popup footer will be used automatically. View this page with the popup masthead.

Official standards & guidelines

ibm.com/standards/web/design/mastheads/


What

A masthead is the highest part of the page that identifies the site you are on, along with navigation elements for the highest level of the site.


Why

Because users want to know what web site they are on. As well, they should be able to easily traverse across major sections and divisions within the company.


What's new

The masthead is all new for v18, focusing on the new categories/divisions/pillars/whatever-you-want-to-call-them.


Rules & guidelines

  • Every page is required to have a masthead.
  • You must never hack or customize the masthead. Classes or HTML may change behind the scenes that will cause your code to be invalid.
  • Any page can choose to use the alternate masthead.
  • Web applications must use the alternate masthead.
  • If your page uses the menu nav for site navigation, you must use the alternate masthead.
  • Popup windows must use the popup masthead.

How

There are three types of mastheads. To use a masthead other than the default, use the standard method for setting a page configuration in v18. Choose the type of masthead that you want on your page:

			<script>
				IBMCore.common.util.config.set({
					masthead: {
						type: "______"	
					}
				});
			</script>

Options for masthead type values are:

  • "alternate"
  • "popup"

The masthead is like a "super module". It's actually a module that contains other modules, each of which have their own configurations, settings and APIs. At a bare minimum you have to have a masthead with the IBM logo. All other items/modules in the masthead can be customized or disabled. You can customize masthead search, search type-ahead, masthead profiling, and even disable them using the standard page config utility. Goto the v18 elements list page and filter on "masthead" to see the various customization options.

The masthead has a "show on scroll up" feature that will make the masthead sticky until the user scrolls back down (enough). This masthead "sticky feature" can be disabled by adding this config to your page:

		<script>
			IBMCore.common.util.config.set({
				masthead: {
					sticky: {
						enabled: false
					}
				}
			});
		</script>

Metrics events

Event that fires on hover of a masthead main item or megapanel trigger item:

  • ibmEV:masthead
  • ibmEvAction:hover
  • ibmEvGroup:$id  (the ID of the item that was hovered on)
  • ibmEvLinkTitle:$title  (the text of the item that was hovered on)
  • ibmEvSection:$type  (the type of menu it triggered: dropdown | megamenu | megapanel trigger | none)
