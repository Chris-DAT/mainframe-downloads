Element classes

  • ibm-parallax-scroll
    Optional class to add to your <div> that automatically sets the background to fixed cover no-repeat.

All examples are using 100% standard compliant elements and classes. The only custom styling that is present is the background styling on the .ibm-band, which is required.

 

Code snippet for above

		<div class="ibm-band ibm-padding-top-0 ibm-padding-bottom-0 ibm-alternate-background" style="background-color: #000;">
			<div data-widget="parallaxscroll" data-scrollspeed="20" style="background: transparent url(https://ibmdev.webmaster.ibm.com/v18/img/bandimages/planetsun.png) no-repeat scroll 100% 30% / contain ;">

The example above is for demonstration purposes only, to show that you can have multiple nested <div>s set with parallax scrolling with different backgrounds set to different scroll speeds to give a more 3D effect. Note that the more and larger images you have, the slower the performance of the page will be.

Official standards & guidelines

ibm.com/standards/web/design/parallax/


What

Parallax scrolling is a technique in computer graphics and web design, where background images move by the camera slower than foreground images, creating an illusion of depth in a 2D scene and adding to the immersion.


Why

Parallax scrolling effects can help liven up a section and provide a nice subtle element of depth (of field) in otherwise flat 2D imagery.


What's new

The parallax scrolling widget is new and can easily be enabled on any full width band section.


Rules & guidelines

The parallax scroll widget can be enabled on any section that has a background image (lead space or .ibm-band). All rules that apply to imagery and lead spaces apply to .ibm-band and parallax scrolling band sections. Ex: No text in images. Ensuring overlaid text is accessible, readable, etc.
As tempting as it is, don't go crazy with this. The more and larger images you have, the slower the performance of the page will be.


How

Simply take an existing full width band section or create an .ibm-band section and add the data-widget="parallaxscroll" attribute to it. 

		<-- Add other background-xxxxx styles here as appropriate. -->
		<div class="ibm-band" data-widget="parallaxscroll" style="background-image: url(__REPLACE_SOME_IMAGE.HERE);">
			<div class="ibm-fluid">
				....Standard grid columns (.ibm-col-x-x) here....
			</div>
		</div>

As with the full width band section, when you define your background-image style on the <div>, you need to also set the other background properties based on your text and imagery and desired effect for the proper placements. You must set background properties separately because the widget automatically adjusts some of them. If you use the style shorthand background property on your container it will not work correctly.

  • Background position: Set this using "%" or "px" units to position the image properly (horizontal) and test the responsive behavior by resizing your browser. NOTE: The widget changes the vertical position on scroll (that's how it works), so it will override whatever you set for the vertical position. Always assume top position (0/top).
  • Background size: By default this is already set to "cover" for the widget, but you can set to other values such as "contain" based on your needs.
  • Scroll speed: You can adjust the default scroll speed of the widget by adding the data-scrollspeed attribute. The default is 6. It works like a ratio. You are saying "For every ##px the browser scrolls, move my image 1px. For example: data-scrollspeed="20" says "move my image up 1px for every 20px that the page scrolls" (this is slow). If you set data-scrollspeed="2" you are saying "move my image up 1px for every 2px that the page scrolls" (this is fast).
  • Disabling on small screen: Often on a small screen you hide the background image and use a solid background color instead (because there's not a lot of space where you will even see the image.) So to disable parallax scroll widget on small screen you simply add the disablesmallgrid="true" attribute to the widget div and it will prevent the widget from moving the background up on a small screen.

Hints and tips

You can add padding and "air out" the band section by using the standard static or responsive padding classes.

You can get creative and do things like put your entire page in a band but using a subtle faded background or place a background color on your ibm-fluid on top of an image. Or place several bands in sequence one right after the other with the parallax scroll widget on each of them to create that nifty scrolling + wiping effect seen below.
