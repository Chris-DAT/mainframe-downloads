Official standards & guidelines

ibm.com/standards/web/design/loading-spinner/


What

A loading spinner is used to help identify to the user that some action is being processed or in progress.


Why

When a user triggers an action like a file upload or a data refresh that may take more than a couple seconds, it's a good idea to indicate to them that something is happening or that their action is being processed. This will help prevent the user from thinking it's stuck or that they need to repeat the action because it didn't work.


What's new

There is only one class in v18: "ibm-spinner". You can use the spinner either inline as a <span>, or standalone as a <div>. The spinner can be sized using the new standard "ibm-h1,2,3,4" classes. If used inline as a span, the spinner will inherit the size of the font from the parent element where the size class is. If used standalone as a <div>, put the ibm-h# class on the spinner <div> itself to size it.


Rules & guidelines

  • Use it smartly. You don't need to show a spinner for every action that the user does.
  • Use it in cases where the response time of an action may be unknown, or you know it'll take more than a couple seconds to process the request. You can even set a timer to show the spinner after 1.5 seconds or whatever, and then kill the timer and spinner when the processing is complete. That way the spinner only shows if it's been long enough to need to show it.
  • The spinner container cannot have text or anything inside of it. It's meant to be an empty element.

How

There are two ways to make a loading/spinner. Inline and as a stand alone/block element. All you need to do is add a special class.

  1. As a stand alone spinner, make an empty div with the special class "ibm-spinner":

    <div class="ibm-spinner"></div>

  2. Inline with text: The same, just put the class on a span. The span can go anywhere inline with text. It doesn't have to be first:

    <span class="ibm-spinner"></span>
