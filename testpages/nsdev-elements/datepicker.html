Official standards & guidelines

ibm.com/standards/web/design/date-picker/


What

A widget that allows the user to pick a date or time from a calendar or list of times instead of manually entering it.


Why

Date/Time pickers are easier to input a time or date, and can be used to help prevent incorrect formatting as well as restrictions (like certain dates or times that are unavailable).It also provides control over the exact formatting of the date of the field.

Rules & guidelines

The date picker widget contains translation support for some of the IBM locales; currently only translated in these 40+ below. If a specific locale translation is not available, US/EN is used. If there is a specific need and you would like to provide a translation file for a locale not listed, copy the US/EN one, translate it, and email it to me (Michael Santelia): http://www.ibm.com/common../../../js/datepicker/data/usen.js

Hints and tips

Date formatting: Change the default date Display a human-friendly format and use an alternate one to submit to the server.

Date restricting: Change the dates/range in which a user can choose from. When using a small range/date window, is it suggested to also disable the year and month selector because there is no need for it (with small date range options).

Disabling translations: If the translations are not proper or complete for your locale, you can opt-out of the automatic translation feature by adding data-enabletranslations="false" to the widget <input>.

For all usage options, detailed documentation, APIs, and advanced usage read the official date picker documentation.
