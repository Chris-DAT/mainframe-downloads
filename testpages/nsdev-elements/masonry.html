Official standards & guidelines

ibm.com/standards/web/design/masonry/


What

Masonry is a JavaScript grid layout library. It works by placing elements in optimal position based on available vertical space, sort of like a mason fitting stones in a wall.


Why

Sometimes it's just cool. But sometimes you have a real purpose for it, like dynamic layouts for dynamic content. It's especially useful if you are using filtering for your content items (with JS) and need to dynamically re-layout your cards when you add or remove items.


Rules & guidelines

  • You must include the supplemental "masonry.js" file on your page to enable Masonry.
  • Masonry is not restricted to cards. Masonry is particularly useful with any block element: cards, media objects, or any block link.
  • Use the standard ibm-col-x-x as containers for each item so they are sized properly.
  • When using Masonry, do not use the "set same height" widget. Either you're laying out cards in a nice square grid, or like bricks on a wall. You can't do both.

How

  • Do not use multiple rows of columns like you would when manually laying out your grid.
  • Make a single ibm-fluid div and inside it put all of your ibm-col-x-x items.
  • Each ibm-col-x-x should contain one block/item.
  • Add data-widget="masonry" to your columns container that has all of the items in it.
  • Add data-items=".ibm-col-12-4" (or other appropriate CSS selector) to your ibm-fluid container to tell masonry that each column you put in there is an item to shuffle.
  • If you are dynamically adding/removing items (ex: user filtering, lazy-loading items infinite scroll, etc) and need to re-layout the cards, call masonry's layout function on your container and masonry will re-adjust your elements:
    jQuery("[data-widget='masonry']").masonry("layout");   (Change jQuery selector as appropriate for your use)
  • All standard Masonry events, APIs, etc are available. Read the official Masonry documentation for advanced usage.
