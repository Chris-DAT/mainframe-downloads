Element classes

  • ibm-background-<color>-<number>
    Colors the background of the element it's applied to unless the element has a specific style that is not meant to be overridden.
  • ibm-text-box-light
    Colors the element background white with an 80% transparency.
  • ibm-text-box-dark
    Colors the element background black with an 80% transparency.

We use the IBM design language color palette and there are ~140 colors. Use these classes and never code your own background color code so your colors automatically update if/when the design system color palette changes.

Background class convention is: ibm-background-<color>-<number> for example:
ibm-background-red-80
ibm-background-green-10
ibm-background-gray-80
ibm-background-purple-70

Special classes: There are two special background color classes available for text that is overlaid above a busy background. Instead of using a solid-fill color class below, you can use a generic light or dark transparency class:

  • ibm-text-box-light: Makes the background white with an 80% transparency.
  • ibm-text-box-dark: Makes the background black with an 80% transparency.

Official standards & guidelines

ibm.com/standards/web/design/color-palette/


What

Colors, for backgrounds.


Why

Because who doesn't love color?


Rules & guidelines

Deciding what colors to use is up to you. It is also up to you and your responsibility to choose an appropriate text color that passes accessibility for your background color you use: The same rules that would apply to imagery.

Background colors can be used in:

  • Lead space
  • Bands
  • Cards
  • Blocks
  • Blocklink
  • ... and other elements

How

  • Add a color class to the element.
  • Choose an appropriate text color that passes accessibility for the background color you use.
