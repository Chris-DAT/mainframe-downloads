Element classes

  • ibm-haschildlist
    Add to a main menu item <li> that has a menu.
  • ibm-sitenav-menu-name
    Add to the <div> that contains the site nav menu's name (site name).
  • ibm-sitenav-menu-list
    Add to the <div> that contains the site nav menu list (<ul>).
  • ibm-parent-site-name
    Add to the <div> in the masthead, after the logo, that container the parent site's name.
  • ibm-sitenav-menu-container
    Main site nav menu container class on the outer <div>.
  • ibm-highlight
    Add to a main nav item and/or submenu item <li> that you want to show as highlighted ("you are here").
  • ibm-sitenav-menu-item-right
    Add to the rightmost <li> to create an item on the right side of the menu nav. It can contain a plain link, button link, or another dropdown menu.

Right side link option

There is an option to have a link on the right side of the menu navigation. It can be either a plain link, a button, or another stand-alone menu.
To make your last menu item show on the right site, simply add the ibm-sitenav-menu-item-right class to the <li> and use a standard link, button, or same menu that you would use in the menu nav.


Dark theme option

For pages with a black or dark lead space, there is a dark option that essentially reverses the background and text colors; background will be black, text will be white.
To use this new alternate color scheme, add the ibm-alternate-dark class to the ibm-sitenav-menu-container div. Example:

Official standards & guidelines

ibm.com/standards/web/design/navigation/#menu-navigation


What

A basic menubar and menu navigation pattern for your site. It consists of main menu items each having (optional) individual sub-menus. It also contains your site name (optional), linked to your site's home page.

NOTE: The menu nav bar is NOT part of the masthead. Visually it is under the masthead... but so is the lead space. The menu nav bar is "content". It is a div right before the #ibm-leadspace-head. It it not controlled by the core v18 JS. What you code is what you get.


Why

The menu navigation allows users to see the entire list of main nav items and each of their sub-nav items. This differs from tab and left navigation where you can only see the current item's sub-nav. It also allows you to put the exact same navigation snippet on every page and optionally just change what menu items you want highlighted ("you are here"). It removes the navigation element from splitting up the lead space and content space, which in certain cases allows for better flow and readability. The optional site branding frees you to use unique <h1> text and lead space imagery on every page.


What's new

This navigation standard is new for v18, but it's not a new pattern. It's based on a standard application menu. As in... look at your browser's application menu "File, Edit, View...". That's a menu bar with each item having a submenu. That's what this pattern is, but for your site. As well, the menubar is sticky to the top of the browser viewport which enables the user to traverse to another menu page without the need to scroll to the top of the page.


Rules & guidelines

  • Note: You must use the "alternate" masthead config setting on your page when using the menu nav.
  • A menu nav is meant to be an alternative for traditional tab navigation.
  • Site name branding on the left is optional and must be hot-linked to your site's home page if used.
  • Parent site name branding in the masthead is optional and must be hot-linked to your parent site's home page if used.
  • A menu nav can only contain two main levels of navigation similar to tabs and left navigation items.
  • The menu bar is automatically sticky to the top of the browser viewport when the links are visible. Stickiness can be disable by adding the data-sticky="false" attribute to the sitenav-menu container
  • Each main item can optionally have a drop-down menu. If a main item has a drop-down menu, the main item is not hot-linked.
  • If a main item has a menu, that main item cannot and must not be hot-lined. Put an entry for the "main item" in the menu. It's how a menu works.
  • A button can optionally be used in the menu nav, but it must be a button-link, small button. Primary or secondary and any color can be used though.
  • Unlike tabs, the menu nav does not need to have an item highlighted.
  • If you are dynamically injecting the entire menubar module HTML post-load, you will need to call it's init function after you inject it for it to work correctly:
    IBMCore.common.module.sitenavmenu.init();.
    And then to push your sitenav menu into the burger menu, call:
    IBMCore.common.module.mobilemenu.addSiteNavigation();
  • If you are dynamically adding a link in the sitenav, call the IBMCore.common.module.sitenavmenu.setSitenavMinWidth() to reset the point at which they will hide when the screen is too narrow to fit them all.
  • If you have main menu items that conflict with the corporate masthead link items, like "Products", "Solutions", "Industries", etc.. you can provide an alternate text/label to be used in the burger menu. This is only needed if you have direct-conflicting names that would be confusing to the user. Add a data-altlabel="____" attribute with your full text to use to the <button> or <a> that you want to have a different text shown in the burger menu. This page contains an example of that.

How

To make a site menu nav you can copy/start with the example below.

  • If the main item has a submenu, add the "ibm-haschildlist" class to the main item <li>, and the item text should be in a <button>
  • If the main item does not have a submenu then simply hotlink the main item's text to goto the page.
  • To highlight a main nav item and/or a submenu item, add the "ibm-highlight" class to the <li>
  • This is not a list, it's a menu. Therefore all proper aria tagging (@role)s must be coded in the HTML, as shown below.
  • To disable the default sticky functionality, add the data-sticky="false" attribute to the sitenav-menu container.
  • If you are dynamically injecting the menubar module HTML post-load, you will need to call it's init function after you inject it for it to work correctly:
    IBMCore.common.module.sitenavmenu.init();.
    And then to push your sitenav menu into the burger menu, call:
    IBMCore.common.module.mobilemenu.addSiteNavigation();
  • To have the burger menu use different text for the main menu item when it's in the burger menu, add a data-altlabel="____" attribute with your full text to use to the <button> or <a>. This page contains an example of that.

To add the optional parent site link in the masthead next to the IBM logo, copy and modify this HTML and add it after the <div id="ibm-home"> element in the masthead:

<div class="ibm-parent-site-name">My optional parent site name</div>
