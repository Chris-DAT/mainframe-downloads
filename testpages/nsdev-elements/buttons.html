v18 styles

There are 4 ways you can manually control the width of the button:

  • ibm-fullwidth: Sets the width to 100% and will fill whatever container it's in.
  • ibm-width-xx: Sets the width to the specified class increment (seen below).
  • ibm-minwidth-xx: Sets the min-width to the specified class increment.
  • ibm-minheight-xx: Sets the min-height to the specified class increment.

Widths: 32, 48, 64, 80, 96, 128, 160, 192, 256, 320, 384

Min-heights: 32, 48, 64, 80, 96, 128

You can force a width of a button by adding a preset width class on the button element (<button> or <a>).
Icons can only be used with text with 128 class and above.
Proper width usage is the responsibility of the page owner.

Element classes

  • ibm-btn-pri
    Makes the button a primary button (solid colored fill).
  • ibm-btn-sec
    Makes the button a secondart button (transparent fill).
  • ibm-btn-<color>-<number>
    Colors the button. Different colors are available for light and dark backgrounds.
  • ibm-btn-transparent
    Add-on for a secondary button, removes the border so it looks like a plain colored link, on hover the background fills with light gray (no options).
  • ibm-btn-small
    Make a small style button.
  • ibm-btn-nowrap
    Prevents the button text from wrapping, keeping it a single line and truncates text with an ellipsis.
  • ibm-button-link
    Used on a container <p> to make a standard link look like a button (basically gives it a colored rectangle brackground).
  • ibm-btn-tag
    Makes a small version of the button with icon on the right, to be used as a filter/selection tag.
  • ibm-fullwidth
    Makes the button span 100% of whatever container it's in instead of fitting to the button text. Great for cards.
  • ibm-btn-row
    Needed on the container <p> with multiple buttons in a row. Ensures proper spacing.

Button modifier classes


Secondary button

Make any button "secondary" style by using the "ibm-btn-sec" class instead of "ibm-btn-pri".

Code snippet for above

<p><button class="ibm-btn-sec">Some button text</button></p>

Small button

Make any button small by adding the "ibm-btn-small" class to it.

Code snippet for above

<p><button class="ibm-btn-pri ibm-btn-small">Some button text</button></p>

Colored buttons

Color any button by adding a button color class to it. Some colors are for "dark" background use only. You are responsible for ensuring you use a color button variation that will be accessible (color contrast) with your background if your background is other than black or white.

Code snippet for above

<p><button class="ibm-btn-pri ibm-btn-blue-50">Some button text</button></p>

Link button (<a>) with an icon

Turn any normal standalone icon link into a "button" by simply adding the primary button class ibm-btn-pri to the <a>. Any standard icon class and button color class can be used. View the icons page for more info and examples.

Some link text

Code snippet for above

<p class="ibm-ind-link"><a class="ibm-btn-pri ibm-forward-link" href="#__REPLACE_ME__#">Some link text</a></p>

Tag

A tag is a special button used on forms or faceted browse patterns to display current filters or items selected. It requires you to code Javascript for it to "remove" the item, as appropriate for your usage. Single common design, there are no options.

Disabled button

Disable any <button> by adding the standard HTML "disabled" attribute to it.

Code snippet for above

<p><button disabled class="ibm-btn-pri">Some button text</button></p>

Preventing button text wrap

Button text wraps by default and the button height will grow to accomodate. You can force the button to stay a single line height and have the browser automatically truncate the text if it doesn't fit in the container horizontally by adding the ibm-btn-nowrap to any button.

Code snippet for above

<p><button href="##__REPLACE_ME__##" class="ibm-btn-pri">Some button text that is long and I want it to wrap</button></p>

Code snippet for above

<p><button href="##__REPLACE_ME__##" class="ibm-btn-pri ibm-btn-nowrap">Some button text that is long and it will not wrap</button></p>

Full width button

Buttons will size according to the button text. To force a button to be full-width of whatever container it's in, regardless of the amount of text, add the ibm-fullwidth class to it.

Code snippet for above

<p><button href="##__REPLACE_ME__##" class="ibm-btn-pri ibm-fullwidth">Some full width button in a 12-6</button></p>

Aligning buttons with text or icon links

Buttons can also be combined and aligned with plain text and standard icon links. Read the alignments options page for more information and code examples.


Multiple button in a row

For multiple buttons in a row (horizontally), no matter what type, you must add the "ibm-btn-row" class to the container <p> so they space out and fold properly. Buttons in a row automatically size to 100% on small grid.

Code snippet for above

<p class="ibm-btn-row"><button class="ibm-btn-pri">Submit</button> <button class="ibm-btn-sec">Cancel</button></p>

Multiple button in a row - secondary transparent

Sometimes a secondary button is too strong for your secondary action, but making it a plain link is too puny. The new transparent option for secondary buttons fill that gap. When you add the ibm-btn-transparent class to a secondary button, the border becomes transparent so it looks like a plain colored link. However it is still a button, and on hover the background fills with light gray.

For dark background where you need a light colored transparent button add the ibm-btn-light class to it.

Code snippet for above

				<p class="ibm-btn-row"><button class="ibm-btn-pri">Submit</button> <button class="ibm-btn-sec ibm-btn-transparent">Cancel</button></p>

Official standards & guidelines

ibm.com/standards/web/design/buttons/


What

A button is an interface object that you can click to perform an action or command, such as submitting a form. True buttons only exist as an <input> element in forms or the newer <button> element that can be used beyond the scope of a form.


Why

Buttons are used to submit forms and allow the user to perform a user-interface action that does not link to another web page.


What's new

The HTML is the same as v17 for basic default buttons, but new classes, conventions and colors have been added for v18.


Rules & guidelines

Know when to use what type of buttons.

  • <input> elements are only allowed inside <form> elements.
  • <button> elements can also be used inside a <form> instead of an <input> and they can be used outside of <form>. The <button> element functions just like buttons created with the <input> element but they offer richer rendering possibilities: The <button> element can have content whereas an <input> does not. Be sure to read the HTML standards and specs on how to properly use <button> elements.
  • <a> elements (link buttons) should be only be used to make a button that links to a web page.

<button> and <input> elements are not to be used as links (to other pages). Use the standard <a> with the button classes to create a normal web page link that looks like a button.


How

There are 3 properties of buttons that you can modify via classes for each. You can mix and match any of these classes on your <input>, <button>, or <a> to create the button of your choosing.

  1. Type: primary or secondary class:
    • ibm-btn-pri
    • ibm-btn-sec
  2. Color: Various color classes, see "Examples" section.
  3. Size: Normal (default) or small class:
    • ibm-btn-small

Other rules and requirements:

  • For a single <input> & <button> button, simply wrap it in a plain <p>.
  • For a link <a> button, wrap it in a <p class="ibm-button-link"></p>.
  • For multiple buttons in a row (horizontally), no matter what type, you must add the "ibm-btn-row" class to the <p>.
