Element classes

  • ibm-margin-top-(0|1|2|3)
    Adds small, medium, or large top margin to an element.
  • ibm-margin-top-r(1|2|3)
    Adds responsive small, medium, or large top margin to an element. The wider the browser, the larger the margin gets.
  • ibm-margin-bottom-(0|1|2|3)
    Adds small, medium, or large bottom margin to an element.
  • ibm-margin-bottom-r(1|2|3)
    Adds responsive small, medium, or large bottom margin to an element. The wider the browser, the larger the margin gets.
  • ibm-padding-top-(0|1|2|3)
    Adds small, medium, or large top padding to an element.
  • ibm-padding-top-r(1|2|3)
    Adds responsive small, medium, or large top padding to an element. The wider the browser, the larger the padding gets.
  • ibm-padding-bottom-(0|1|2|3)
    Adds small, medium, or large bottom padding to an element.
  • ibm-padding-bottom-r(1|2|3)
    Adds responsive small, medium, or large bottom padding to an element. The wider the browser, the larger the padding gets.
  • ibm-padding-content
    Adds 20px padding on each side. Use for cards, blocks and blocklinks to pad the content away from the edge of the box.
  • ibm-nospacing
    Forces min-height, margin and padding to 0, all around.

The below band uses top and bottom r2 padding. That's 5% of the browser width. When the browser is 1600px wide that makes 90px vertical padding. That looks good on desktop and allows me to have a lot of room to show off this nice big background image. However on smaller screens 90px vertical padding is wasting a lot of screen real estate. As well, my image does not have the same effect because the screen is so small, and I'd probably want to just hide it anyways. So using the r2, when the browser scales down to 400px wide that makes 20px vertical padding which is perfect.

On nice big monitors that have a lot of real estate I get to show off this big image. But at the same time without having to code anything special I can make that container shrink down to a more appropriate size for the browser width it's being viewed on.

Squeeze your browser down to see the top and bottom of the band start to close in on the text.

Official standards & guidelines

ibm.com/standards/web/design/padding-classes/


What

Classes you can use for more vertical space between things.


Why

Sometimes you want to air things out and show off cool background imagery. Or sometimes you have huge text you want more space between.


Rules & guidelines

Margins and padding, along with text sizes, were created as a prescribed system of measurements that work together as a cohesive system. They weren't randomly chosen. Use the standard margin and padding classes.


How

There are two types of margin and paddings: static and responsive.

Static spacing classes are fixed PX heights: 20, 40, 60px. Use these when you always want a fixed spacing between things no matter what the browser width.

Responsive spacing classes are variable heights. The units are % of the browser width: 2, 5, & 10%. How does this work? Let's do some math: If the browser is 1000px wide, then 10% of that == 100px. That same class when viewed on a smaller screen, say 500px == 50px. Responsive spacing work well in lead spaces and bands or areas where you things spaced out on a large screen, but not a small screen. 60px bottom spacing is OK on a desktop that's got a browser height of 800px. But on a small screen that's only 500px high... 60px is a lot of space to take up with nothing.

In the chart below:
The green lines represent the static spacing class (1, 2, 3). As the browser grows in width, the spacing never changes.
The red lines represent the responsive spacing classes (r1, r2, r3). Since they are based on a % of the browser width, the wider the browser is the larger the spacing amount grows.


Hints and tips

Spacing classes can be stacked/nested to give you even more spacing. And you can get creative and mix and match static and responsive. Doing that allows you to say "I want at least 20px minimum spacing on this, but as the screen grows wider, I want the spacing to increase from there up." Get creative people.
