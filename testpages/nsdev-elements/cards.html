Element classes

  • ibm-card
    Required outer container class to make/setup a card style element.
  • ibm-card--noborder
    Removes the outer card border, best used with cards with colored backgrounds
  • ibm-card__heading
    Adds 20px inner padding, used for an optional card heading section.
  • ibm-card__image
    Does nothing anymore, deprecated.
  • ibm-card__content
    Same has __heading; Adds 20px inner padding, used for the content section of the card.
  • ibm-card__bottom
    Use to bottom-align a CTA. Best used with set same height widget with multiple cards.
  • ibm-padding-border-left
    For use with 10px left border variant. Sets left padding to 30px to account for the 10px spacing lost by the pseudo left border design. Can be used on card or blocklink.

A card is really just a "blocked" piece of content typically with a border. That's it. It does not need to have an image, contain specific elements, nor be any specific column width.

Official standards & guidelines

ibm.com/standards/web/design/cards/


What

Simply put, cards are pieces of content with a border around it with the same/similar pattern. Much like a baseball card.


Why

A card is a way to present a piece of content in a concise, easy-to-scan style, while by virtue of being a "block" makes it work well with RWD (responsive web design).


Rules & guidelines

  • Use the proper ibm-card class and HTML.

How

  • Use the code snippets below for examples of cards.
  • Make sure to use the proper divs and new classes to properly space and layout your card.
  • You can use any v18 border color for your cards. Gray is the default.
  • You can use any v18 background color for your cards.
  • No border option: Depending if/what your background color is, the cards might be better off without a border. To use the no-border option, add the class ibm-card--noborder to your <div class="ibm-card">.
  • Bottom align CTA option: You can align your CTA links by putting them in the ibm-card__bottom section and using the setsameheight widget and it's data-always="true" option on your cards. The special bottom section is bottom aligned in the card which allows it to align with other cards. Note that if your CTA text/item spans multiple lines and others don't, then they will not line up (obviously because heights are differently).

Hints and tips

  • Don't bother using an image if it does not add meaning to your content.
  • Cards are best used as 2-4 across.
  • Use the same card design for all cards in a section. Mixing card designs impales scanability.
  • Use the set same height widget on a row of cards to force the cards to be the same height, regardless of content.
  • Common elements used in any combination (but not required) as needed based on content and design:
    • Border
    • Background color
    • Heading
    • Image
    • Main content
    • Some CTA (link or button)
