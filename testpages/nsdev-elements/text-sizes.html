Element classes

  • ibm-h1
    Sizes the text as an h1 heading.
  • ibm-h2
    Sizes the text as an h2 heading.
  • ibm-h3
    Sizes the text as an h3 heading.
  • ibm-h4
    Sizes the text as an h4 heading.
  • ibm-fontsize-body
    Sizes the text back to the default body font size.
  • ibm-small
    Sizes the text smaller than the default body font size.
  • ibm-medium
    Modifier class to add with the ibm-h1 class to go up one step from the h1 size.
  • ibm-large
    Modifier class to add with the ibm-h1 class to go up two steps from the h1 size.
  • ibm-fontsize-x25
    Will size the text 2.5x whatever size it's currently styles as. Best used for data/#s visualizations.

2.5x multiplier

There is a 2.5x multiplier available. This is a little advanced usage, so you need to understand how it works. The special class "ibm-fontsize-x25" will multiply whatever class's size you are using by 2.5 or 250%. The class cannot be applied directly to the same element you are sizing (because only 1 font size can apply per element). It has to be applied to a child /nested element. For example:

This is a paragraph with .ibm-h2 class that has a child span that has the ibm-fontsize-x25 class

Code snippet for above

<p class="ibm-h2">This is a paragraph with .ibm-h2 class that has a child <span class="ibm-fontsize-x25">span that has the ibm-fontsize-x25 class</span></p>

Official standards & guidelines

ibm.com/standards/web/design/text/


What

Sizes for both heading and normal text.


Why

Because sometimes size is needed to show hierarchy, and sometimes size is simply eye candy.


Rules & guidelines

Be smart and use sizes wisely. It's not a game of how many different sizes you can use on one page. It's a game of "what size makes sense here". It you're not sure how to use sizes effectively, consult a designer or UX professional. Please.

All elements, including headings (H1-6) are set to the default body font size. Now, you're thinking "Umm that makes no sense."... but it does. Here's why we did it: To allow a separation between the proper semantic heading tag that should be used vs. the visual representation of that text desired for the design. Basically the design is not tied to the semantic meaning. Yes, this is debatable and ideally using a hierarchy of H1-6 sizes enforces semantics through design. But this is reality and with current design trends, doing it this way allows for flexibility and more importantly allows you to tag your content correctly and not be forced to mis-use headings or use an inappropriate heading simply due to the desired design.

In order to size any paragraph or heading element to be larger than the default, you must add a size class. You can combine size classes with text colors, text weights, and text alignments to format your text any way you want.


How

  • Type some text.
  • Add a size class to the element.
  • Like text color classes, size classes can be used on any element anywhere on the page.
