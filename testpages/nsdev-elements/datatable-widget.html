Official standards & guidelines

ibm.com/standards/web/design/data-tables/


What

The data table widget allows you to add additional functionality to a basic table, such as sorting, filtering, pagination and more.


Why

When you have large sets of data to show users it's helpful to let them control the data to read it the way they want. This could me a different sort order, letting them filer only rows that they want to see, etc.


What's new

The widget is new for v18, but the plug-in used (datatables.net) encompasses and allows for both v17 functions: scrolling (horizontal) and sorting, and there's a lot more you can do.


Rules & guidelines

The only rule is to add the proper widget attributes to your table to use any particular or combination of plug-in options you want.


How

The table uses all standard IBM data table HTML, but has special data-xxxx attributes added to the table element which control the widget. To enable the widget, you must add the data-widget="datatable" attribute to your <table>, and then any of the following attributes can be used.

  • data-col-reorder="true" Enable the user to reorder the display of the columns
  • data-info="true" (Shows "Showing 1-10" texts)
  • data-ordering="true" (Enables sorting)
  • data-paging="true" (Enbles pagination)
  • data-scrollaxis="x" (Allows horizontal scroll)
  • data-searching="true" (Enables text filtering/searching)

To view more details on the options and advanced plug-in functionality, view the data table widget JS documentation.


Additional JS/CSS files required

Required supplemental JS and CSS files for this element:

<script src="https://1.www.s81c.com/common../../../js/tables.js"></script>
