Element classes

  • ibm-alternate
    Add this class to the ibm-leadspace-head element to make the lead space a shorter height.
  • ibm-leadspace-fluid
    Due to legacy support, you must add this class to the ibm-leadspace-head to enable the fluid 12-column grid to work in the lead space. Also requires the grid-fluid.css file on the page.
  • ibm-video-background
    Add this class to the ibm-leadspace-head element when using the lead space video background option container ibm-video-background-container.

The examples below are made with varying combinations of standard classes for; text colors, sizes, weights, alignment, background hiding, responsive and static padding, and more. Do NOT use any of the images below on your pages. They are for placement/demonstration purposes only.

NOTE: To use the 12-column fluid grid in the lead space, you must add the ibm-leadspace-fluid class to the ibm-leadspace-head element so that it allows the space to be 100% width.

Official standards & guidelines

ibm.com/standards/web/design/lead-spaces/


What

The lead space is the area immediately under the masthead. It signifies the start of the page "content". It typically contains titles, images, or important call-to-action links.


Why

The lead space is the top of the content space that the user sees immediately when the page loads. It's the place to make a first impression and set the tone/messaging of the page so use it wisely.


What's new

In v18, there are no requirements or rules. And the lead space is no longer required. You can do everything you could do in a lead space in an ibm-band now. Therefore consider the ibm-leadspace-head and ibm-leadspace-body classes deprecated. You can omit the lead space entirely on your page and simply put your <h1> in the content body.

If you do use the ibm-leadspace, because of RWD (responsive web design), it now requires the standard grid and columns for proper layout and can use any element that can be used in the content space. It's no different from the full-width band section.


Rules & guidelines

Because of html5 and RWD, the rules have changed for the lead space in that there are very few rules now.

  • Optional: The lead space element "ibm-leadspace-head" is now optional. If you aren't using it as a lead space and just have basic page title/subtitle in there, there's no need for it.
  • Required: If you are using the lead space container, the <div id="ibm-leadspace-head"> and <div id="ibm-leadspace-body"> wrapper classes are required.
  • Required: Use the standard column grid in the lead space for proper layout and responsive design.
  • Optional: Nav trail/Breadcrumbs.

How

  • Required div IDs: Use the same standard <div id="ibm-leadspace-head"> and <div id="ibm-leadspace-body"> wrappers from v17.
  • To automatically make all the text white: Add the the ibm-alternate-background class to the <div id="ibm-leadspace-body"> if using a dark background. This is a convenience class and text color classes will not work inside an element with this class.
  • The ibm-alternate-background class can be applied to any parent element to change text and links inside it to white.
  • To make a short (height) lead space: By default the lead space is tall. Add the "ibm-alternate" class to the <div id="ibm-leadspace-head"> to make it shorter (smaller min-height).
  • The content can be laid out using any of the standard grid columns options.
  • Any of the standard elements can be used inside it.
  • Lead spaces can can be formatted using any of the standard background colors, text sizing, text colors, text weights or alignment options and combinations.
  • You can spice up your lead space by adding advanced features like parallax scrolling and animation but don't go crazy. Keep it classy and meaningful and remember that animations take computing power and do not stop when out of view. They keep animating.
