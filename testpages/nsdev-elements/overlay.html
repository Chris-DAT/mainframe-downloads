Element classes

  • ibm-common-overlay
    Base class required on an overlay <div> container.
  • ibm-overlay-alt
    Add this class to make it a medium size overlay.
  • ibm-overlay-alt-two
    Add this class to make it a large size overlay.
  • ibm-overlay-alt-three
    Add this class to make it a x-large size overlay.

There are two types of overlays:

  1. The typical modal/dialog which can be used for any basic content and the user can close it/exit out without "doing" anything.
  2. An alert overlay which locks the overlay and forces the user to explicitly take some sorta action in the overlay content space in order to close it. It requires the additional data-type="alert" attribute on the container div. It cannot be closed by any other means. Whatever your "close" action is, you must pass true to the standard overlay hide function to close an alert type overlay, like: overlay.hide(true).

Official standards & guidelines

ibm.com/standards/web/design/overlays/


What

An overlay is a basic content container laid over the page content that can be used for asking questions, obtaining input, introducing features, indicating progress, giving instructions, or revealing information.


Why

An overlay can be used as a type of selective disclosure to only show the user auxiliary content as needed.


What's new

While the v17 overlay HTML works, it has been deprecated. Simplified overlay HTML has been setup for v18.


Rules & guidelines

  • Overlays are meant to be a lightweight content container. Do not shove content into an overlay that is better suited to be directly "on the page".
  • Columns cannot be used in an overlay.
  • Interactive widgets should not be used and are not meant to be used in an overlay.
  • Basically, if you've got more than some simple text, or buttons, or basic fields, you're doing it wrong and you should consider an alternate pattern like an on-page show/hide section.
  • If you need to force the user to explicitly take action on the overlay content, add data-type="alert" to the overlay container. This will "lock it" and you must then control/allow for the closing of the overlay. Call the standard overlay hide API, but pass an/additional "true" param to force the alert overlay to close.

How


Basic overlay HTML

To create an overlay widget container, simply wrap the content in: <div class="ibm-common-overlay" data-widget="overlay" id="__someID__"></div>.

Example:

		<div class="ibm-common-overlay" data-widget="overlay" id="__someID__">
		    
This is my overlay content

		</div>

Overlay sizes

Overlays have five size options available. The default size is small:

  • Small (390px)
  • Medium (570px): Add class ibm-overlay-alt
  • Large (750px): Add class ibm-overlay-alt-two
  • Xlarge (980px): Add class ibm-overlay-three
  • * Full width (browser window): Add class full-width

* Full width overlay is special use and content is centered by default. Optionally with the full-width size, you can add the seamless class to reduce the inner padding. This is best used when the overlay contains an image only (see bottom example).


How to show an overlay

Overlays are shown just like v17: An overlay can be shown from any type of link, button, or programmatically via JS. Simply call the overlay show function, passing it the ID of the overlay you want to show. Example:

__SOME_LINK_TEXT__

Advanced use

Overlays have several APIs available including show/hiding and dynamically changing the HTML content. And as with any widget, they can be programmatically created with JS (passing the settings via an object). Read the JS doc for full information.

Metrics events

Event that fires when tabs are loaded and when one is clicked indicating which tab was activated/shown:

  • ibmEV:widget
  • ibmEvAction:$id  (the ID of the overlay that was shown/hidden)
  • ibmEvGroup:Overlay
  • ibmEvName:$event_type  (the event that happened; show | hide)
