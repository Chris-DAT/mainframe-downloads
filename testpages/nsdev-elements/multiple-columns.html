Element classes

  • ibm-textcolumns-<#>
    For use on a <p/ul/ol> to automatically split it into # columns across, like a newspaper. Uses CSS column-count.
  • ibm-textcolumns-<grid_size>-<#>
    Allows you to override the columns count originally set for the text, for each size grid, except on small grid. Small grid only has a two-column split (next class).
  • ibm-textcolumns-small-2
    Allows you to split the text on a small size grid so it's split in two text columns instead of the default one.

Official standards & guidelines

ibm.com/standards/web/design/text-alignment/


What

The CSS3 multi-column layout allows easy definition of multiple columns of text - just like in newspapers.


Why

Long lines of text and small font or spacing can be hard to read. This gives you control to dynamically change that by telling the browser to split up your paragraph into multiple (virtual) columns. Also awesome for lists, especially dynamic content lists, and even moreso when using filtering. You don't need to manually layout/split the list into columns.


Rules & guidelines

  • It is a CSS style for plain text and lists... things that are not "block" elements.
  • It will only work on plain text nodes <p> and lists <ul/ol> etc.
  • The browser does the splitting/layout for you. NOTE: Depending on the # of items vs. columns you want, list items may not be split evenly across all columns. This is how the browser works.
  • On the small screen grid all classes are automatically set to a single column, just like ibm-col-#-# is. You do not need to specify a column split for small grid (because it will do nothing).
  • Text can be split across up to six columns.

How

The classes work in the same fashion as the grid. You set the default class for how many columns you want the text to be split across, and then you can optionally use grid-override classes to change the column split for different size grids. For example: On a medium screen you might want text or a list split into 2 columns, but on a x-large screen split into 4 columns (since there's more horizontal real estate.

The class convention to split your paragraph or list across columns is:

  • ibm-textcolumns-1
  • ibm-textcolumns-2
  • ibm-textcolumns-3
  • ibm-textcolumns-4
  • ibm-textcolumns-5
  • ibm-textcolumns-6

The number at the end of the class is how many (virtual) columns to split your text into. The override classes are the same as for columns. The naming convention is: ibm-textcolumns-<size>-#. Where you replace <size> with the grid size you want to set a different span for:

  • xlarge
  • large
  • medium
  • small
