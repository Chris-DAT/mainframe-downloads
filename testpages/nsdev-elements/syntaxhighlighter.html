(Never view the page source or use Firebug to copy HTML. Only use provided code snippets.)

Use optional @class values to show line #s and highlight lines. Class value attributes are via CSS syntax:   <property>:<value>

  • brush: js (default/dynamically added if no @class)
  • html-script: true (default/dynamically added if no @class) - highlights HTML code if it exists.
  • gutter: true (default is false) - shows line numbers if set to true.
  • highlight: ## - Highlights a specific line #. Use an array to highlight multiple lines. Ex: highlight: [1,2,5,6,7]

All examples below use the same code snippet, but simply have different settings on the widget.

Default syntax highlighter, no extra options set:

	<head>
		
		
		
		
		<script>
			IBMCore.common.util.config.set({
				someProperty1string: "string value",
				someProperty2bool: false,
				someProperty3arr: ["val1", "val2"],
				someProperty4int: 23432
			});
		</script>
	</head>

Official standards & guidelines

ibm.com/standards/web/design/syntax-highlighter/


What

A code syntax highlighter that enhances the <pre> element by adding color coded text as well as optional line #s and the ability to highlight specific line(s). Allows the user to easily highlight the entire code block and copy it by double-clicking it.


Why

Sometimes you want more than just a basic monotype font to display a code snippet, as well, make it easier to read and scan by using color-coding.


Rules & guidelines

  • Use a standard <pre> element with the proper widget and optional class values.
  • You may have to escape your opening tags, depending what kind of code you are displaying, to prevent it from rendering your code sample.
  • Code snippet lines do not wrap so as to preserve exact code snippet and line #s.

How

  • Place your code inside a pre element with the required "data-widget" attribute: <pre data-widget="syntaxhighlighter"> </pre>
  • Default "brush" and "html-script" options are dynamically added if no @class exists on the pre.
  • If you use either the optional "gutter" or "highlight" options you must also include the "brush" and "html-script" options.
  • Showing line #s and highlighting a line can be used independently of each other.
  • You can enable vertical scrolling (helps with tall code snippets and responsive design) by using the max height option: @data-maxheight="###"

Additional JS/CSS files required

Required supplemental JS and CSS files for this element:
