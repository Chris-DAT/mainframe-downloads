New left navigator

The left navigator for v18 has twisty/show-hide functionality built into it allowing sub-navs to be expanded and collapsible, like a basic show/hide.

Existing left navs: The HTML for existing left nav element is the exact same and the same rules apply and it will look and function the exact same as it has been in v16/17/18 as-is. All existing coded left navs have simply received a visual design update.

Enabling new v18 left navs: To enable the new collapsible sub-navs the HTML is a very minimal change. When the main nav item acts as a show/hide trigger control, it cannot be a link. Therefore, for your main item that contains a sub-nav, you would place the text inside a <span class="ibm-subnav-heading"> instead of a <a>.

For example, instead of this:

			
  • 
				Downloads
				<-- sub nav UL here -->

    • The new v18 HTML for that item should be:

    			
  • 
				Downloads 
				<-- sub nav UL here -->

    • This also means that the link that you would have placed as that main item, you now have to make as a link in the sub-nav instead. Because obviously you can't have an item onclick be both a show/hide control and act as a link to a page.

    This special sub-nav heading span is the "flag" that the JS looks for to turn that main item and it's sub-nav into a collapsible show/hide sub-nav. Otherwise it will just display inline as a list, as left nav sub-navs have been.

    Expanding a sub-nav by default: When you are on a page and you want a sub-nav item to be selected as the "you are here" item, you can make that sub-nav be expanded by default by adding the aria-expanded="true" attribute to the <li> of the main item you want expanded.

    Essentially, this is just like the new site menu navigator... you can have a single piece of HTML/fragment on all pages and the only thing you need to change is which item is "selected". No more of this "only code the sub-nav for the page you're on" business.

    Since you can only have one left nav per page, the below examples will not display correctly, but the HTML is correct.

    Official standards & guidelines

    ibm.com/standards/web/design/navigation/#left-navigation


    What

    The special left nav page layout is now deprecated, and you can now make a "left nav" in the standard landing page/tab-nav layout, allowing you more freedom with the grids and elements you can use.


    Why

    So you have more flexibility


    What's new

    The HTML is the same, just put it in whatever column span you want. Typically a 5-1 with a 6-2 on "medium" grid. See example below. And your content goes in the other 5-4 and 6-4 next to it. You can put bands and use the full 6 col grid under the left nav if you choose.


    Rules & guidelines

    Same as v17 rules but you need to add proper accessibility tagging. The styling will not work properly without it. For example, to show an item as selected, you must use the aria-selected="true" attribute.


    How

    Use the official v18 standard template generator to create a proper, compliant, standard v18 page shell with a left nav. Then just take that left nav <nav> element and put it in a column on a normal page layout with leadspace, etc.
    NOTE: Use the ibm-hidden-small class on the 5-1 column that houses the left nav to hide it on small grids, otherwise you will end up with a big list of links at the top of your page on a small grid (which maybe you are OK with if there are only a few of them.)
