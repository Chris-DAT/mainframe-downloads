Element classes

  • ibm-vertical-tabs
    Enables the dynamic tabs to be displayed in a vertical stack instead of traditional horizontal row.
  • ibm-tab-section
    Formats the tab list container.
  • ibm-tabs
    Formats the <ul> as a row of tabs.
  • ibm-tabs-content
    Tells the widget that this is a container of a tab's contents.
  • ibm-graphic-tabs--colored
    Used on the outer <div> container to color graphic tabs. Sets colors to inherit and allows standard background, text, and border color classes to color the tabs <ul>.
  • ibm-background-<color>-<number>
    Use standard background color classes to color dynamic graphic tabs background.
  • ibm-textcolor-<color>-<number>
    Use standard text color classes to color dynamic graphic tabs text.
  • ibm-border-<color>-<number>
    Use standard border color classes to color dynamic graphic tabs underline.

Dynamic tabs - vertical option

These are the same as the standard dynamic tabs above, can be colored the same as below, and have all the same rules.
The difference is that the tabs <ul> goes in one column and the content in the other space/column span - as opposed to the horizontal tabs which have the tabs and the tab content on top of each other in the same column.

The only HTML difference is to add the special ibm-vertical-tabs class on the div with ibm-graphic-tabs so they stack vertically.

The tabs can be placed in any size column span as appropriate and fits your needs/content. Tip: Use the column grid override classes to fine-tune your tab set width.

Official standards & guidelines

ibm.com/standards/web/design/navigation/#tabs-navigation


What

Dynamic tabs are simply a different form of show/hide. They are basically just horizontal tabs that show and hide the content associated with each tab.


Why

A different form of show/hide other than vertically stacked "+"s.


What's new

Tabs' backgrounds and borders can be colored using any of the standard v18 color palette classes.


Rules & guidelines

These should not be used in lieu of proper page navigation tabs. Shoving pages worth of content into show/hides will dilute your SEO efforts.

  • Both type of graphic tabs; primary navigation (the ones under the lead space) and content space tabs can be turned into dynamic tabs by adding the widget to them.
  • Each tab must have a corresponding <div id="____" class="ibm-tabs-content"></div> in order to function properly.

How

  • Code your tabs using the standard HTML for the ones you're using (primary nav or content space graphic tabs).
  • Enabling dyntabs: Add the data-widget="dyntabs" attribute to the outer tabs container.
  • Instead of linking the tab to a page, make it a page anchor link using the #. That hash value you set will correspond to the ID of it's content container.
  • Content space tab contents: Create a <div id="____" class="ibm-tabs-content"></div> for each tab, putting it's contents in it. You must use the tab's hash value as the value of the div's id.
  • Because you can have multiple sets of content space tabs on the page, all of the content tabs' <div id="____" class="ibm-tabs-content"></div> must be a child element to the tab set they belong to.
  • This is a lot harder to explain in words. Just look at the example snippets and you'll immediately understand it all.
  • Default showing tab: By default the first tab is highlighted. You can enable the widget to show a different dynamic tab by default/on page load by setting @data-activetabid="____" on the tab container and put the hash value of the tab you want highlighted/shown by default if other than the first one.
  • Tab memory: Each set of tabs has "memory" and will remember the last tab you were on for up to 4 hours (or whatever it's currently set to).
  • Disabling tab memory: If for whatever reason you don't want the tabs to remember the user's last state and to always highlight the default/first one on every page view, add the data-remembertab="false" attribute to your tab widget container.
  • Linking to a page's tab: You can link to a particular tab on a page (have it shown by default), by adding the hash value to the page URL you're linking to. The syntax is <some_page_url>#tab_xxxxx, where "xxxxx" is the ID of the tab you want to show.
  • Colored tabs: Graphic tabs can be colored by adding the required ibm-graphic-tabs--colored class to the ibm-graphic-tabs container. Then you can then change the background and bottom border color of the tab set by adding any of the standard background colors, border colors, and text colors classes to the tab <ul>..

Metrics events

Event that fires when tabs are loaded and when one is clicked indicating which tab was activated/shown:

  • ibmEV:widget
  • ibmEvAction:$id  (the ID of the tab that was clicked or shown onload)
  • ibmEvGroup:Dynamic tabs
  • ibmEvName:$event_type  (how the tab was shown; load | click)
