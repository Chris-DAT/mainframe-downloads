Element classes

  • ibm-tabs
    Formats the <ul> as a row of tabs.
  • ibm-graphic-tabs
    Used on the outer <div> container to style the tabs like traditional blocks tabs.
  • ibm-text-tabs
    Used on the outer <div> container to style the tabs like plain text tab links.
  • ibm-graphic-tabs--colored
    Used on the outer <div> container to color graphic tabs. Sets colors to inherit and allows standard background, text, and border color classes to color the tabs <ul>.
  • ibm-text-tabs--colored
    Used on the outer <div> container to color text tabs. Sets colors to inherit and allows standard background, text, and border color classes to color the tabs <ul>.
  • ibm-background-<color>-<number>
    Use standard background color classes to color graphic tabs background.
  • ibm-textcolor-<color>-<number>
    Use standard text color classes to color graphic tabs text.
  • ibm-border-<color>-<number>
    Use standard border color classes to color graphic tabs underline.

Three types of navigation tabs

  1. Primary tab & subtab navigation: Tabs as primary navigation, special div and placement above the content space..
  2. "Graphic" tabs: As primary navigation at the top, in the content space.
  3. "Text" tabs: As primary navigation at the top, in the content space. Or secondary navigation to a left navigator

Tab nav layout: Primary navigation tabs

Look up at the bottom of the lead space. That's them. Primary and secondary (sub) tabs. Same as v17 but ensure you have proper accessibility tagging. Rules are the same as v17.

NEW: Primary tabs #ibm-primary-tabs now have a "sticky" option. View the sticky tabs option page for information.

NOTE: If you have to many tabs to fit on one line for primary tabs (graphic), you can use secondary tabs (text tabs) as your tab navigation instead. In other words, omit the ibm-primary-tabs and use ibm-secondary-tabs for your tab navigation. This means you cannot have "subtabs" because you are technically using "subtabs" as your primary navigation.

Official standards & guidelines

ibm.com/standards/web/design/tabs-navigation/


What

Tab-based navigation consists of buttons or tabs aligned side by side and it is usually located at the top of the page. Tabs should use limited as well as short and predictable titles.


Why

Navigation tabs help the users know where they are and find their way around the web site. Tabbed navigation is a good choice when the number of tabs is not likely to change often and the tabs fit on one row.


What's new

Tabs can be colored using any background and border color.


Rules & guidelines

Rules are the same as v17: Primary tabs and graphic tabs must only have enough tabs that fit on one line. Read the v17 standards for the complete set of rules.


How

  • Primary tabs should be created using the template generator.
  • Use code snippets below as starters for left nav secondary navigation tabs.
  • Colored tabs: Graphic tabs can be colored by adding the required ibm-graphic-tabs--colored class to the ibm-graphic-tabs container. Then you can then change the background and bottom border color of the tab set by adding any of the standard background colors, border colors, and text colors classes to the tab <ul>.
  • Primary tabs #ibm-primary-tabs have a "sticky" option. View the sticky tabs page for more information.
