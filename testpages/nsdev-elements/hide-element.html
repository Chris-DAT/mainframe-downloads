Element classes

  • ibm-hidden-small
    Sets the element to display: none when the small grid is being shown.
  • ibm-hidden-medium
    Sets the element to display: none when the medium grid is being shown.
  • ibm-hidden-large
    Sets the element to display: none when the large grid is being shown.
  • ibm-hidden-xlarge
    Sets the element to display: none when the x-large grid is being shown.
  • ibm-show-small
    Only shows the element on the small grid. Will be hidden on all others.
  • ibm-show-medium
    Only shows the element on the medium grid. Will be hidden on all others.
  • ibm-show-large
    Only shows the element on the large grid. Will be hidden on all others.
  • ibm-show-xlarge
    Only shows the element on the x-large grid. Will be hidden on all others.
  • ibm-mobile
    DEPRECATED: Do not use. Use standard ibm-hidden-xxx classes instead. This will be removed soon.
  • ibm-no-mobile
    DEPRECATED: Do not use. Use the standard ibm-hidden-small class instead. This will be removed soon.

You can apply the classes to any element; a table, a link, a paragraph, an entire row of content, whatever. For demonstrative purposes I have tagged each image below to hide on a particular grid.

Official standards & guidelines

ibm.com/standards/web/design/hide-images/


What

Classes you can add to any element to hide it on a particular size grid.


Why

In v17 there used to be just "mobile" or "no mobile", but in 7 years a lot has changed. Now there are thousands of screen sizes not just "mobile" or "desktop". These allows you to further refine what element you want to display on the screen on any of the grids.


What's new

Instead of "no-mobile" and "mobile-only", the classes are now matched to our standard programmatic grid sizes.


How

The classes are similar and correspond to the the hide background classes:

  • ibm-hidden-xlarge
  • ibm-hidden-large
  • ibm-hidden-medium
  • ibm-hidden-small
