Element classes

  • ibm-blocklink
    Add this to an <a> to make it a block-level link with any type of content. Use with ibm-padding-content to give it proper inside padding. A block link acts like a div and can contain any element (except a child <a>). You can even hotlink an entire band.
  • ibm-highlight-light
    Add this to a block link to give it a light border on hover instead of the default blue one. Use on block links inside dark background bands.
  • ibm-highlight-dark
    Add this to a block link to give it a dark border on hover instead of the default blue one. Use on block links inside light background bands.
  • ibm-highlight-bg-<color>-<number>
    Alternate highlight/hover effect. Add one of these classes to make the block link background (and text) change color instead of the border on hover . Highlight color classes pass a11y with light and dark text and text color is automatically changed accordingly.
  • ibm-padding-content
    Adds 20px padding on each side of the element. Use to pad the content away from the edge of the block link box.
  • ibm-padding-border-left
    For use with 10px left border variant. Sets left padding to 30px to account for the 10px spacing lost by the pseudo left border design.

Official standards & guidelines

ibm.com/standards/web/design/block-links/


What

Link a "block" of content, instead of linking just inline text.


Why

Because in html5 an <a> is allowed to contain block-level elements and previously it wasn't, so now you can.


Rules & guidelines

  • Block links have transparent border by default.
  • "Default state" border can be colored using standard border color classes.
  • Default hover styling is border color change. Optional default hover classes based on your background/needs are ibm-highlight-light and ibm-highlight-dark
  • An alternate hover style can be used that changes the entire background color (and text) of the block link instead of the border on hover.

How

The block link is not an element, but more of a generic modifier than you can apply to various standard elements. While it might work for almost anything, there are obvious places and elements where it doesn't belong so use some common sense. To make content into a block link, you wrap it in <a href="__REPLACE_ME__" class="ibm-blocklink"> ... </a>.

You then put your element/content inside the are normal. Remember, don't code your link inside that because that is your link. If you want to make text inside your block link look like a link use the ibm-icon-nolink and ibm-linkcolor-default class (or other color class if non-white background, as appropriate) to make your text look like a link.

A perfect example of this is the first example. You'll see that the text that looks like a link, actually isn't a link, it's just text styled to look like a link. In case you're wondering "HUH"... in HTML you can't put a link inside a link. It doesn't work.
